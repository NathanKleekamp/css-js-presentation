The Web Experience in the Autistic Spectrum

Speaker: Natalia Berdys


Web accessibility: We usually picture someone with a physical disability.

The Web Experience in the Autistic Spectrum

  • Structure: provide sitemap, search, breadcrumbs, group nav in one place.
  • Copy: avoid idioms and imperatives, write clear titles and summaries, use lists, don't justify text because “rivers” are distracting.
  • Design: Avoid sounds, animations, bright colors, patterns.

Wrap up

A lot of this stuff is really new. And won't be ready for general use in a year or two.

One thing is very clear: Browser innovation is happing faster and faster.

  1. The Extensible Web Manifesto
  2. Polymer and Brick projects polyfilling web components so that devs can play a more active role in shaping browser APIs.
  3. There will be no "CSS4" because individual modules will be upgraded independently and itteratively.
  4. ECMAScript is moving to an annual release cycle.

State has to do a better job upgrading it's browsers.

Because anyone at State who uses a major service on the Internet will be effected, especially PD folks.

Chrome should be our default browser
